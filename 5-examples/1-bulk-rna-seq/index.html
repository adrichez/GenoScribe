Contexto y pipeline empleado

Este informe presenta un análisis integral de datos de RNA-Seq (mRNA, miRNA y circRNA) mediante el uso del pipeline miARma‑Seq (Eduardo Andrés-León et al.), una herramienta modular, escalable y fácil de usar que integra múltiples pasos clave en un flujo de trabajo reproducible. Este pipeline está diseñado para funcionar con cualquier organismo con genoma secuenciado y combina herramientas bioinformáticas ampliamente utilizadas en un marco automatizado.

Resumen del flujo de trabajo de miARma‑Seq:

  1. Control de calidad inicial: Evaluación de los datos crudos mediante FastQC, permitiendo detectar adaptadores, sesgos de composición y problemas de calidad por posición.
  2. Recorte de lecturas: Eliminación de adaptadores y bases de baja calidad utilizando herramientas como Cutadapt o Reaper, mejorando la calidad de los datos antes del alineamiento.
  3. Alineamiento: Mapeo de lecturas contra el genoma de referencia utilizando alineadores como HISAT2, STAR, Bowtie o BWA, según el tipo de datos.
  4. Cuantificación de entidades: Cuantificación de genes o isoformas mediante featureCounts, detección de circRNAs con CIRI y cuantificación de miRNAs con miRDeep2.
  5. Análisis diferencial: Identificación de genes o miRNAs diferencialmente expresados usando paquetes como edgeR o NOISeq, según el diseño experimental.
  6. Análisis funcional: Predicción de interacciones miRNA‑mRNA, evaluación de correlaciones inversas de expresión y análisis funcional de enriquecimiento en términos de Gene Ontology (GO) y KEGG.

El diseño modular de miARma‑Seq permite ejecutar el pipeline completo de forma automática o seleccionar pasos individuales según las necesidades del usuario. Esta flexibilidad lo hace especialmente adecuado para ser integrado en sistemas de análisis reproducibles como Nextflow, donde cada módulo puede definirse como un proceso independiente.

En este análisis, automatizado mediante Nextflow y presentado con Quarto, se parte de una carpeta raíz generada por miARma‑Seq, conteniendo los resultados estructurados de cada paso. El objetivo de este informe es resumir visualmente y de forma comprensible cada etapa, desde el control de calidad inicial hasta el análisis funcional, proporcionando una herramienta clara y reproducible para la interpretación de datos transcriptómicos.

A continuación, se muestra una representación esquemática del flujo de trabajo que sigue el pipeline miARma‑Seq para el análisis de datos de RNA-Seq y miRNA-Seq. Este diagrama ilustra claramente la naturaleza modular del pipeline, permitiendo observar cada etapa del proceso, desde el control de calidad inicial hasta el análisis funcional de los resultados.

El pipeline se estructura en diferentes módulos, cada uno de los cuales desempeña una función específica:

  • Módulo de calidad: Utiliza FastQC para evaluar la calidad de las lecturas crudas en formato FASTQ. Genera informes HTML que permiten detectar problemas comunes como adaptadores, baja calidad o sesgos.
  • Módulo de recorte (opcional para miRNA): Herramientas como Cutadapt y Reaper eliminan adaptadores y regiones de baja calidad, especialmente relevantes en miRNA-Seq.
  • Módulo de alineamiento: Dependiendo del tipo de RNA, se emplean diferentes alineadores:
    • BWA para circRNAs.
    • Bowtie1/Bowtie2 para mRNAs y miRNAs.
    • miRDeep2 como alternativa específica para miRNA.
  • Módulo de cuantificación: featureCounts permite cuantificar la expresión génica a partir de los archivos alineados (BAM). Para circRNAs, se usa CIRI.
  • Análisis diferencial: El pipeline permite realizar análisis de expresión diferencial mediante edgeR y NOISeq, adaptándose a la disponibilidad de réplicas o diseño experimental.
  • Módulo funcional: Finalmente, se lleva a cabo la anotación funcional y análisis de enriquecimiento usando GOSeq, permitiendo vincular los resultados con rutas KEGG y términos GO.
  • Predicción de objetivos: En el caso del análisis de miRNAs, se incluye un paso de predicción de interacciones miRNA‑mRNA mediante miRGate.

Este flujo de trabajo proporciona una visión clara y organizada del enfoque integral del análisis, combinando flexibilidad, eficiencia y reproducibilidad, elementos clave que se trasladan directamente al sistema automatizado desarrollado en este informe.

Para garantizar la reproducibilidad y modularidad del pipeline, miARma‑seq utiliza un archivo de configuración con extensión .ini. Este archivo permite especificar todos los parámetros necesarios para ejecutar el análisis, sin necesidad de escribir comandos complejos en la línea de comandos.

El archivo .ini se estructura en secciones bien definidas (por ejemplo, [General], [Aligner], [ReadCount]…), cada una dedicada a un módulo del análisis. En cada sección, se indican los valores clave a utilizar: rutas de entrada y salida, tipo de datos, número de hilos, herramientas a emplear, criterios estadísticos, etc.
